# Perubahan v0.2.0 - 2026-03-10

## Ringkasan
- Implementasi seluruh backend Python: config, history, video cloner (FFmpeg), API bridge untuk pywebview, dan entry point aplikasi.

## Perubahan

### File Baru
- `src/config.py` — Class `Config` untuk load/save konfigurasi JSON.
  - Default values: ffmpeg_path, clone_count, format, template, output_folder, method, notifikasi.
  - Method: `load()`, `save()`, `get()`, `set()`, `update()`, `to_dict()`.
  - Path default config: `config.json` di root proyek.
- `src/history.py` — Class `History` untuk menyimpan riwayat clone dalam JSON.
  - Setiap entry memiliki UUID unik, source_file, duration, clone_count, method, format, timestamp, elapsed_total.
  - Method: `load()`, `save()`, `add()`, `get_all()`, `clear()`.
  - Path default history: `history.json` di root proyek.
- `src/cloner.py` — Class `VideoCloner` + fungsi utilitas.
  - **VideoCloner**: threading-based clone dengan progress tracking.
    - Fast method: video copy + audio re-encode dengan noise imperceptible + metadata unik.
    - Standard method: full re-encode CRF 17-23 (random) + noise filter video + audio noise.
    - Randomisasi parameter per clone: audio bitrate, noise volume, CRF, padding metadata.
    - Auto-rename jika file output sudah ada (suffix angka).
    - Progress: status, current_index, percent, elapsed, estimated_remaining, items per-clone.
    - Callback: `on_progress`, `on_complete`, `on_error`.
    - Cancel support: kill FFmpeg subprocess.
  - **get_video_info()**: informasi video via ffprobe (filename, size, duration, resolution, codec, bitrate, fps, audio info).
  - **check_ffmpeg()**: deteksi ketersediaan dan versi FFmpeg.
- `src/api.py` — Class `Api` sebagai bridge Python ↔ JavaScript (pywebview).
  - File dialog: `select_video_file()`, `select_output_folder()`, `select_ffmpeg_path()`.
  - Google Drive download: `download_from_gdrive()` dengan cookie handling, virus scan bypass, progress tracking.
  - TikTok download: `download_from_tiktok()` via tikwm.com API (no watermark).
  - Instagram Reels download: `download_from_reels()` via yt_dlp.
  - YouTube Shorts download: `download_from_ytshorts()` via yt_dlp.
  - Facebook Reels download: `download_from_fbreels()` via yt_dlp.
  - Cloning control: `start_cloning()`, `cancel_cloning()`, `get_clone_progress()`.
  - Config management: `get_config()`, `save_config()`.
  - History management: `get_history()`, `clear_history()`.
  - FFmpeg check: `check_ffmpeg()`.
  - Window controls: `minimize_window()`, `toggle_maximize()`, `close_window()`.
  - Utility: `open_folder()`, `play_notification_sound()`, `get_full_path()`.
- `src/main.py` — Entry point pywebview (1280x800, frameless, easy_drag=False).
- `config.json` — File konfigurasi default.
- `history.json` — File riwayat clone (awalnya kosong).
- `requirements.txt` — Dependency: pywebview>=5.0.

### File Pendukung
- `src/test_check.py` — Script utilitas untuk cek ffmpeg, config, dan webview.

## Dampak
- Backend siap digunakan: semua API tersedia untuk dipanggil dari frontend JavaScript.
- Mendukung 5 sumber video: file lokal, Google Drive, TikTok, Instagram Reels, YouTube Shorts, Facebook Reels.
- Clone menghasilkan file yang binary/hash berbeda meskipun visual identik.
- Progress real-time dan ETA tersedia untuk monitoring.
