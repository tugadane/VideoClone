# Perubahan v0.6.1 — 2026-05-05

## Ringkasan
Patch UX: memperjelas dukungan Shopee (dan YouTube Shorts) di Batch Links. Fungsionalitas batch Shopee sebenarnya sudah berfungsi sejak v0.6.0 (logika `detectSourceFromUrl`, `SOURCE_CONFIG.shopee`, dan `download_from_shopee` semua sudah ada), tapi placeholder Batch Links hanya menampilkan TikTok / Instagram / Facebook sehingga user tidak tahu Shopee bisa dipakai di sini juga.

## Perubahan
- `src/web/index.html`
  - Placeholder textarea `#inputBatchLinks` diperbarui:
    - Header info: "Platform is auto-detected: TikTok, Instagram, Facebook, YouTube Shorts, Shopee, Google Drive".
    - Tambah dua contoh URL: YouTube Shorts (`https://www.youtube.com/shorts/XYZ`) dan Shopee (`https://id.shp.ee/abcd1234`).

## Tidak ada perubahan logic
Tidak ada perubahan di `cloner.py`, `api.py`, `app.js`, atau file lain selain hint UX di placeholder.

## Verifikasi
Trace logic di app.js:
1. `parseBatchLinks()` → memfilter baris yang diawali `http://` / `https://` (link Shopee lolos).
2. `detectSourceFromUrl(url)` → mengembalikan `'shopee'` untuk `shp.ee/` atau `shopee.<region>/`.
3. `SOURCE_CONFIG.shopee` → memetakan ke `download_from_shopee`.
4. `download_from_shopee()` di api.py mengembalikan info dengan `source_platform: 'shopee'`.
5. `onFileSelected(info)` → source masuk ke `sourceFiles` dengan tag platform → badge tampil → auto-watermark Shopee aktif saat clone.
