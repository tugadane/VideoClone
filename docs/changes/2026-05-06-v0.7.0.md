# v0.7.0 — Image Overlay + 3 posisi vertikal-tengah baru

**Tanggal:** 2026-05-06
**Tipe:** minor (fitur baru)

## Ringkasan

Menambahkan **Image Overlay** sebagai panel baru sejajar dengan Text Overlay dan Video Overlay, dan **memperluas dropdown posisi di ketiga overlay** dengan tiga opsi baru: **Center Left**, **Center Center**, **Center Right**.

## Perubahan UI

### Image Overlay (panel baru, optional + multiple)
- Kolom baru di area Output, struktur identik dengan Text Overlay & Video Overlay (collapsible, header + tombol "Add Image Overlay").
- Tiap card image overlay punya:
  - File picker (Browse) — menerima `*.png`, `*.jpg`, `*.jpeg`, `*.webp`, `*.bmp`, `*.gif`
  - Slider **Overlay Size** (5–100% lebar video utama, default 25%)
  - Dropdown **Position** (9 opsi penuh; default `bottom-left`)
  - Slider **Opacity** (10–100%, default 100%)
- Bisa tambah lebih dari satu image overlay sekaligus, dikombinasikan bebas dengan Text + Video Overlay.
- Phone preview kanan menampilkan placeholder image overlay dengan border kuning agar mudah dibedakan dari Video Overlay (border hijau dashed kalau chroma key) dan Text Overlay.

### Position dropdown (Text + Video + Image overlays)
Dropdown posisi sekarang punya **9 opsi penuh** (sebelumnya 5–7):

| | Left | Center | Right |
|--|------|--------|-------|
| Top    | top-left | top-center | top-right |
| Center | **center-left** | **center-center** | **center-right** |
| Bottom | bottom-left | bottom-center | bottom-right |

Tiga opsi yang dicetak tebal adalah opsi baru di v0.7.0. Dropdown Text Overlay sebelumnya sudah punya 7 (top-* + center + bottom-*), Video Overlay punya 5 (top-left/right + bottom-left/right + center), Image Overlay sebelumnya tidak ada — sekarang ketiganya punya 9 opsi yang sama.

## Perubahan Backend

### `src/cloner.py`
- Bagian video overlay setup di-refactor: variabel `vo_indices` dan `has_video_overlays` digabung jadi satu list `all_overlays` yang berisi tuple `(input_index, overlay_config, kind)` dengan `kind ∈ {'video','image'}`.
- Image overlays di-append sebagai input dengan flag `-loop 1` (gambar di-loop terus selama durasi video utama; `-shortest` membatasi durasi).
- Filter chain `overlay` dirakit sama untuk video dan image, dengan `format=yuva420p` ditambahkan otomatis untuk image (untuk preserve alpha PNG).
- `chromakey` hanya berlaku untuk video overlay (image overlay sudah punya alpha native).
- Position map `ov_pos_map` diperluas dari 5 jadi 10 entri (9 posisi baru + 1 alias `center` → center-center untuk kompat lama).

### `src/cloner.py` — Text Overlay Y handling
- Logika `base_y` diperluas: `position == 'center' or position.startswith('center')` → `vertical center`. Sebelumnya hanya cocok dengan `position == 'center'`, sehingga `center-left` / `center-right` salah jatuh ke bottom alignment.

### `src/api.py`
- Method baru `select_overlay_image()` (file dialog dengan filter PNG/JPG/JPEG/WebP/BMP/GIF) sejajar dengan `select_overlay_video()` yang sudah ada.

## Kompatibilitas

- **Backward compatible.** Field `text_overlays`, `video_overlays`, dan field `position` value `'center'` lama tetap dipanggil. Tidak ada migrasi data yang dibutuhkan.
- Job yang sebelumnya pakai 5 posisi Video Overlay (top-left/right, bottom-left/right, center) tetap jalan tanpa perubahan.
- File config / history yang diserialisasi sebelum v0.7.0 tetap valid karena field `image_overlays` opsional (default `null`).
