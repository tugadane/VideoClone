# Perubahan v0.6.2 — 2026-05-05

## Ringkasan
Bugfix patch untuk batch download Shopee (dan platform lain): menangani caption yang berisi emoji / karakter non-ASCII, dan memberi error message yang lebih informatif untuk URL Shopee yang bukan halaman video (misal link produk/etalase).

## Bug yang Diperbaiki

### "Downloaded file is not a valid video" pada video Shopee dengan emoji di caption
**Gejala**: Saat batch download 11 link Shopee, 4 di antaranya gagal dengan pesan "Downloaded file is not a valid video", padahal video aslinya bisa diakses dan dimainkan via browser.

**Root cause**: 3 dari 4 video tersebut punya caption berisi emoji (mis. `lucu bgt 😍`). Caption dipakai sebagai bagian dari nama file MP4 yang disimpan ke disk. Saat `ffprobe.exe` dipanggil dari Python untuk validasi metadata, argumen baris perintah dikonversi dari UTF-16 ke system codepage Windows (CP1252 di Indonesia/Western). Emoji `😍` (U+1F60D, supplementary plane) di luar BMP, tidak bisa dipetakan ke CP1252 → jadi `?` → ffprobe tidak menemukan file → return None → "not a valid video".

**Fix**: helper `_safe_filename(text, fallback, max_len)` baru di `src/api.py` yang membersihkan teks ke ASCII printable saja (`\x20-\x7E`) sebelum dipakai sebagai bagian nama file. Emoji + karakter non-ASCII otomatis dibuang. Diterapkan untuk:
- `download_from_tiktok` (gantikan regex sanitizer lama yang tidak strip emoji)
- `_shopee_download` (gantikan regex sanitizer lama)
- yt-dlp options di `download_from_reels`, `download_from_ytshorts`, `download_from_fbreels` — tambah `'restrictfilenames': True` (built-in yt-dlp untuk strip non-ASCII).

### Pesan error generic untuk link non-video Shopee
**Gejala**: Link `s.shopee.co.id/...` yang resolve ke halaman produk (bukan halaman video) gagal dengan pesan generic "Failed to download Shopee video. Make sure the link is valid and the video is public." — tidak jelas kenapa.

**Fix**: `_shopee_download` sekarang return tuple `(filepath, error_reason)` dan menambah deteksi eksplisit:
- Final URL bukan `sv.shopee.<region>` dan bukan `/share-video/` → "Shopee URL bukan halaman video (kemungkinan link produk/etalase, bukan Shopee Video)."
- Halaman tidak punya `__NEXT_DATA__` → "Halaman Shopee tidak berisi data video..."
- `mediaInfo` ada tapi `watermarkVideoUrl` kosong → "Halaman Shopee tidak punya URL video..."
- Network error fetch share page / CDN download → pesan dengan exception message sebenarnya.

## Catatan Bug Sampingan
Karakter non-ASCII yang sah untuk filename Indonesia/Latin (mis. `é`, `ü`, `ñ`) juga terkena strip. Untuk audiens Indonesia ini umumnya tidak masalah; trade-off vs. menghindari bug ffprobe codepage di Windows.
